django流程:
    客户端请求(浏览器url)------>django服务器处理请求的url---服务器提取请求的内容request--->django应用处理请求request------>
    django应用再将处理请求后的结果返回给django服务器------>django服务器将结果返回给浏览器


1.创建django项目/django应用
    1.0.url说明(视图views.py还会有详细说明):
        1.0.1.url相当于---->http://ip地址:port端口/参数
            绝对url:  http://ip地址:port端口/参数       ----->浏览器第一次与服务器，建立联系时使用
            相对url:  /参数                            ----->在服务器内部使用，django框架会自动将其补充为绝对url
            eg：
                http://127.0.0.1:8080/index    # index为参数
                http://127.0.0.1:8080/admin/mine  # admin/mine为参数
        1.0.2.url配置:(结合应用视图部分的笔记)
            1)配置项目中的urls.py(匹配客户端传来url中的参数)---->配置应用中的urls_handle.py(匹配，项目匹配过后的url)
                特别注意:
                    2.1.1)应用中的urls.py需要手动自行创建
                    2.1.2)项目中必须使用urls.py文件(文件名只能为urls.py),应用中的urls_handle.py文件可以使用其他文件名(eg:urls_app.py)
            2)urls.py中的urlpatterns列表中元素的写法(url函数/include函数在django.conf.urls中):
                2.1)url(正则表达式,视图函数名)                    ====>一般用在应用的urls文件中
                        如果正则表达式中有分组，会自动将分组匹配的结果，作为参数传递给视图函数
                2.2)url(正则表达式,include(应用中的urls_handle文件))     ====>一般用在项目的urls文件中
                特别注意:
                   除已经配置好的urlpatterns外,不要使用django.urls中的path函数
        1.0.3.url设计问题(在html模板文件中,链接标签),url重定向的问题:
            统一规范:
                写相对url时前统一加上反斜杠"/"
                eg：
                    在访问 http://127.0.0.1:8080/index 页面时点击某些按钮跳转到某些页面，这些按钮的url设计问题(html文件中的链接标签)
                    /create     ---->相当于http://127.0.0.1:8080/create
                    /add        ---->相当于http://127.0.0.1:8080/add
                    /delete     ---->相当于http://127.0.0.1:8080/delete
                    反例:
                    Append      ---->相当于http://127.0.0.1:8080/indexAppend
                    Kill        ---->相当于http://127.0.0.1:8080/indexKill
        1.0.4.url匹配获取参数(获取的是从浏览器请求的url获取的参数)(联系应用视图部分的笔记)
            0)views.py中的视图函数:
                必须有一个request参数,用来接受请求的环境信息;
                若要接受其它参数,其它参数一定在request参数之后。
            url(re正则,view视图函数)
                方法:对re正则表达式进行分组
            1)参数获取
                0)参数的来源与去向:
                    浏览器的url生成的参数--->服务器程序
                1)url位置参数()
                    re正则中的所有分组，按顺序传递给view视图函数
                        eg:
                            url("^index/(\d+)$",index)      ====>(\d+)匹配到的部分会作为index视图函数的第二个参数
                2)url关键字参数
                    re正则中进行分组并命名，分组的命名需要与view视图函数中的关键字参数的名字一致
                        eg:
                            url("^index/(?P<keyname>\d+)",index)      ====>(\d+)匹配到的参数传给index视图函数中的keyname关键字参数

    2.1.模型类(modeles.py):
        2.1.1.
            1)模型类生成表:

            2)模型类的属性(相当于数据表的字段):
                from django.db import models
                创建字段名 = models.字段类型(选项)
                    几种常见类型:
                        整数:IntegerField
                        字符串:CharField
                            特殊选项:max_length规定字符串的最大长度
                        日期:DateField
                        小数:DecimalField
                            特殊选项
                                max_digits数的总位数，包括小数位
                                decimal_places书的小数位数
                        文本:TextField
                            有4000个字符串大小
                        布尔类型:BooleanField
                            只有True和False
                    几种常见选项:
                        是否为空:null
                            默认为False,表示可以为空，生成表时约束字段（一般不使用）
                        空白:blank
                            默认为False,校验数据时不允许出现空值，传值时约束值（一般不使用）
                        默认值:default
                            用于设置默认值
                        主键:primary_key
                            用于设置主键，一般创建一个模型类时会自动创建一个名为"id"的主键
                        唯一:unique
                            为True时，表示该字段中的数据不能出现重复
                        索引:db_index
                            为True时，为字段创建索引
                        选择:choices
                            规定取值的范围，可以为列表、元组，相当于C语言中的枚举类
                        字段名称:db_column
                            有该选项时用该选项的字符串为表的字段名，否则以变量为字段名
                            eg:
                                inter = IntegerField(defult = 0)
                                以“inter“为表的字段名，默认值为0
                        on_delete一般配置为:models.CASCADE
                            on_delete=models.CASCADE
                3)模型类关系:
                    两个模型类：
                        一般的主要关系是一对多
                        3.1.)一对多(外键):
                            在多类中定义外键:字段名 = models.ForeignKey(模型类名称的字符串)
                                外键对应外键表中的主键，一般是id
                            eg:
                                图书--->图书中的人名
                        3.2.)多对多:
                            在两个类中定义都可以:字段名 = models.ManyToManyField(模型类名称的字符串)
                            eg:
                                新闻---->新闻类型
                        3.3.)一对一:
                            在两个类中定义都可以:字段名 = models.OneToOneField(模型类名称的字符串)
                            eg:
                                学生---->学号
                    一个模型类:
                        自关联:相当于特殊的一对多
                            字段名 = models.ForeignKey("self")
                            自动关联自己表的id字段
                4)让模型类创建指定名称的数据表(指定字段名见字段选项)
                    方法:
                        在模型类对象中创建Mate类:
                                class Meta:
                                    db_table = '表名字符串'  # 指定表名
        2.1.2.
            1)模型管理类:
                用于管理模型，相当于替换掉查询时的objects
                作用:
                    1.1.)改变查询结果
                    1.2.)添加额外的方法
            2)实现方法:(在模型管理类的内部，可以使用self指向被哪一个模型类使用)
                from django.db import models
                class 模型管理器名(models.Manager):                 #   模型管理类会有objects的所有属性

                class 模型类名(models.Model):

                    模型管理属性名 = 模型管理类名                    #   模型管理属性名一般为objects
                #   使用(在.py文件或python manage.py shell中)
                模型类名.模型管理属性名.模型管理的相关函数
                eg:
                    class TempManager(models.Manager):      #   模型管理类
                        def tp(self):
                            return self.get(id=1)           #   self指向Temp
                    class Temp(models.Manager):
                        name = CharField(max_length = 30)
                        obj = TempManager()                 #   obj指向模型管理类,同时将自己的地址传递
                    #   在python manage.py shell中
                        ChinaInfo.obj.tp().name             #   会显示出id=1对应的name属性，相当于没有模型管理类时ChinaInfo.objects.get(id=1).name

    2.3.views.py视图:
        (视图的作用:用于处理请求并返回结果给django服务器)
        1)编写视图函数
            视图函数中必须要有request参数，用来接受http头的数据
            request参数的属性:
                (1) request.scheme
                    请求的协议种类（http/https）
                (2) request.path
                    请求页面的完整路径（get_full_path()在path的基础上还有查询条件）
                (3) request.method
                    请求使用的http方法（POST，GET，DELETE，PUT...）
                (4) request.content_type
                    请求的MIME类型
                (5) request.GET/request.POST
                    get/post请求中的所有参数，类似于字典的对象
                (6) request.COOKIES
                    请求包含的cookie字典
                (7) request.FILES
                    上载文件的类字典对象
                (8) request.headers(Django2.2新功能)
                    获取请求headers里面的内容，不区分大小写，类似字典的对象
                    request.headers["Cotent-Length"]  请求报文中实体主体的字节大小
                    request.headers["Content-Type"]  请求的MIME类型
                    request.headers["User-Agent"]  请求的用户代理
                (9) request.META
                    包含所有HTTP头部信息的字典
                    CONTENT_LENGTH –请求正文的长度（以字符串形式）
                    CONTENT_TYPE –请求正文的MIME类型
                    HTTP_ACCEPT –响应可接受的内容类型
                    HTTP_ACCEPT_ENCODING –响应的可接受编码
                    HTTP_ACCEPT_LANGUAGE –可接受的响应语言。
                    HTTP_HOST –客户端发送的HTTP Host标头
                    HTTP_USER_AGENT –客户端的用户代理字符串。
                    REMOTE_ADDR –客户端的IP地址。
                    REMOTE_HOST –客户端的主机名
                    SERVER_NAME –服务器的主机名
                    SERVER_PORT –服务器的端口（以字符串形式）
                    SERVER_PROTOCOL –http协议版本（HTTP/1.1）
                (10) request.user
                    当前登录用户的实例
                (11) request.session
	                代表当前会话的可读可写，类似于字典的对象（删除用户登录的session，request.session.delete(session_key)
	        request方法:
                (1) request.get_host()
                    获取主机域名或者IP加端口号
                (2) request.get_port()
                    请求端口号
                (3) request.is_secure()
                    如果使用的是Https，则返回True，表示连接是安全的
                (4) request.get_full_path()
                    带有请求参数的完整path

        2)配置url(修改urls.py中的urlpatterns列表)
            2.1)配置项目中的urls.py(匹配客户端传来url中的参数)---->配置应用中的urls_handle.py(匹配，项目匹配过后的url)
                特别注意:
                    2.1.1)应用中的urls.py需要手动自行创建
                    2.1.2)项目中必须使用urls.py文件(文件名只能为urls.py),应用中的urls_handle.py文件可以使用其他文件名(eg:urls_app.py)
            2.2)urls.py中的urlpatterns列表中元素的写法(url函数/include函数在django.conf.urls中):
                2.2.1)url(正则表达式,视图函数名)                    ====>一般用在应用的urls文件中
                        如果正则表达式中有分组，会自动将分组匹配的结果，作为参数传递给视图函数
                2.2.2)url(正则表达式,include(应用中的urls_handle文件))     ====>一般用在项目的urls文件中
                特别注意:
                   除已经配置好的urlpatterns外,不要使用django.urls中的path函数
            eg:
                客户端请求   http://127.0.0.1/hhhindex
                项目中的urls匹配      \hhh
                应用中的urls匹配      index
                客户端中匹配时http://127.0.0.1/hhhindex----->项目中匹配时/hhhindex（将匹配剩下的丢给应用）----->应用中匹配时index
                2.2.3)url捕获(获取url参数，在urlpatterns列表中)
                    传递位置参数:分组(用括号括起来)
                    关键字参数:(?P<参数名>参数值)
            eg:
            urls处理文件中:
                urlpatterns = [
                    url("index/(\d+)",index),               #   位置参数,把(\d+)匹配到的部分传给index(request,num1)中的num1
                    url("center/(?P<num>\d+)",center),      #   关键字参数，参数名为num,将\d+匹配到的传给center(request,num)中的num
                ]
            views.py文件中:
                def index(request,num1):
                    return HttpResponse(num1)
                def center(request,num):
                    return HttpResponse(num)
        3)视图中的函数返回值,传递给html模板文件:
            HttpResponse()返回html对象:
                #    效果和以下步骤的效果一样，request:视图函数中的request参数；template_name:html文件的名称；context:渲染html的参数字典(模板上下文)
                render(request,template_name,context)
                使用方法:
                    from django.shortcuts import render                # 导入(html)渲染方法
                    from django.http import HttpResponse               # 导入HttpResponse对象
                    from django.template import loader,RequestContext  # 用来处理模板文件(处理html文件)
                    from .models import *                              # 导入模板中的模板对象
                    # 1.获取模板
                    load_get = loader.get_template(template_name)      # template_name:模板(html)文件名
                    # 2.渲染获取html文件
                    re_result = load_get.render(template_dict)         # template_dict:模板中的参数字典
                    # 3.html返回
                    return HttpResponse(re_result)                     # 返回模板(html)文件
                HttpResponseRedirect()返回一个页面(也称为“重定向”):
                    使用方法:
                        from django.shortcuts import redirect              # 导入(html)渲染方法
                        from django.http import HttpResponseRedirect       # 导入HttpResponse对象
                        from .models import *                              # 导入模板中的模板对象
                        # 1.其他操作(一般式模板对象操作)
                        # 2.地址返回，返回一个地址，“重定向”
                        return HttpResponseRedirect("url地址")              # 返回url地址
                        return redirect("url地址")                          # 返回url地址
                    注意:
                        redirect和HttpReaponseRedirect效果一样，二者取其一即可
                        render和HttpResponse选其一
                JsonResponse()返回js文件:
                    该返回方式一般用于ajax(页面部分刷新中)
        4)浏览器数据的传输方式:
            GET：请求的数据在“http://ip:port/参数”中，安全性较低
            POST：请求在http数据的头部，安全性较高
        5)cookie与session存储用户的数据:
            cookie:一个字典，数据存储在浏览器中，安全性较低
            session:一个字典，数据存储在服务器中，安全性较高
            cookie与session的举例:
                eg:在view.py中
                    #   cookie
                    #   /set_cookie
                    def set_cookie(request):
                        "设置浏览器的cookie"
                        #设置一个cookie信息
                        response = HttpResponse("set_cookie:--->num:{0}".format(str(1)))
                        response.set_cookie("num",1,max_age=7*24*3600)
                        #返回response
                        return response

            2.4)模板过滤器
                过滤器其实就是python函数
                2.4.0)基本语法：
                    value会自动作为一个参数传递给filter_name过滤器
                    {{ value|filter_name:参数 }} value：变量名 ； filter_name：Django内置或者自定义filter方法； 参数：处理的值
                2.4.1)内置过滤器
                value为模板变量，value会制动作为一个参数传递给过滤器
                    {{ value|default:参数 }} 如果value没有传值，替换default设置的参数。
                    {{ value|default_if_none:参数 }} 如果value为none，替换default设置的参数。
                    {{ value|length }} 显示value的长度
                    {{ value|filesizeformat }} 将value格式化一个可以识别的大小（例子：1.5k，5M等）
                    {{ value|slice:" 2:1" }} 对value进行切片（支持负数）
                    {{ value|date:"Y:m:d H:i:s"}} 格式话时间
                        eg:
                            {{nowtime|date:"Y年-m月-d号"}}         年-月-号
                            {{nowtime|date:"Y年:m月:d号"}}         年:月:号
                    {{ value|safe }}  直接解析为可执行的html代码
                    {{ value|truncatechars:30 }} 对value进行截取，只显示30个字符，用“...”代替多余的部分
                    {{ value|add:10 }} 做加法运算
                    {{ value|capfirst }} 首字母大写
                    {{ value|cut:'a' }} 从字符串中移除指定的字符
                2.4.2)自定义过滤器
                特别注意:
                    2.4.2.1.)自定义过滤器必须要在应用目录下创建一个名为templatetags的包,必须时包而不是文件夹
                    2.4.2.2.)包的名字必须为templatetags
                    2.4.2.3.)在templatetages目录下任意创建一个名称的.py文件
                        2.4.2.3.1.)自定义过滤器必须要有一个参数,并且最多只能有两个参数
                        2.4.2.3.2.)自定义过滤器必须遵循以下格式:
                            格式1:
                                from django import template
                                #   创建一个Library类的对象
                                register = template.Library()     # 只能叫register
                                @register.filter                  #  必须使用register.filter修饰
                                def 函数名(filter_avg):           # 至少有一个参数,最多传两个参数
                                    return  值                    #   必须要有返回值
                            格式2:
                                from django import template
                                #   创建一个Library类的对象
                                register = template.Library()     # 只能叫register
                                def 函数名(filter_avg):           # 至少有一个参数,最多传两个参数
                                    return  值                    #   必须要有返回值
                                register.filter(name="函数名")     #  注册过滤器函数

    2.5.django模板类查询，模板类表格查询:
        模板类对象:----->视为mysql的表格
        模板类属性:----->视为mysql表格的字段
        django查询:
            1)查询函数的语句:    模型类对象名.objects.查询函数(模型属性名__条件名=条件值)
                1.1)模型属性名__条件名=条件值  ----> 条件
                1.2)有时没有条件名
                        有条件名是必须使用两个下划线"__"连接
                1.3)在下面的详细说明中都是通用的
            2)查询函数(一般返回对象都为QuerySet(查询集)对象，有多条查询数据):
                all---->返回所有满足条件的数据
                filter---->返回满足条件的数据
                exclude---->返回不满足条件的数据
                order_by---->对查询结果排序，默认从小到大
                aggregate---->使用聚合函数时使用
                get---->返回满足条件的一条，并且只能是一条数据
            3)条件名（参照第一条的说明）:
                比较查询:
                    gte---->greate than大于等于
                    gt---->大于
                    lte---->less than小于等于
                    lt---->小于
                    qte---->equal等于
                模糊查询:
                    contains---->查询字段包括条件值
                    endswith---->查询字段以某个条件值结尾
                    startswith---->查询字段以某个条件值开头
                范围查询:
                    in---->范围查询,条件值可以是数组、元组,查询值是列表中的某个元素
                空查询:
                    isnull---->判断字段是否为空
                时间查询:
                    year/month/day---->年/月/日查询
            4)条件（参照第一条的说明）:
                条件除了以上写法外还可以使用F对象和Q对象进行操作
                F对象和Q对象的导入:
                    from django.db.models import F,Q
                F对象---->用于类属性间的比较,F对象传的是名字,是一个字符串
                Q对象---->用于条件之间的逻辑关系(与:& 或:| 非:~)，Q对象传的是字段，不是字符串
            5)聚合函数:
                5.1)聚合函数的导入:
                    from django.db.models import Sum,Count,Avg,Max,Min
                5.2)聚合函数传递的是字段对应的字符串，传递的是一个字符串
                5.3)聚合函数的查询必须使用aggregate查询函数
                Sum---->求和
                Count---->计数
                Max---->最大值
                Min---->最小值
                Avg---->平均值
            6)关联查询(含外键的查询):
                6.1.)格式:
                一类(作为外键)对多类的查询:
                    变量 = 一类模型类名.objects.get(条件)
                    变量.多类模型名_set.all()
                6.2.)通过多类的条件查询一类的数据:
                    一类模型名.objects.filter(多类模型名小写__多类模型的属性__条件)
                6.3.)通过一类的条件查询多类的数据:
                    多类模型名.objects.filter(关联属性__一类属性名__条件)
                eg:
                    #   查询id=1的图书(BookInfo)中所有人物的名字(HeroInfo)
                    b = BookInfo.objects.get(id=1)              #   获取id=1的图书
                    b.heroinfo_set.all()                        #   获取id=1的图书，中所有人物的名字
                    #   查询书中有"杨过"的所有书(hname为人物表中存储任务名的字段)
                    BookInfo.objects.filter(heroinfo__hname = "杨过")
                    #   查询id = 1书中所有人物(hbook为人物表中的外键)
                    HeroInfo.objects.filter(hbook__id = 1)

        eg:
            #   book模型类的类属性:id->编号(默认自带)，name->书名，sup->点赞量，com->评价数，dt->发行日期
            from .models import Book
            from django.db.models import F,Q,Sum,Avg
            from datetime import date
            Book.objects.filter(id_gt=10)                           #   书的id大于10
            Book.objects.filter(name__contains='老')                #   书名中有‘老’字
            Book.objects.filter(name__startswith='新')              #   书名以‘新’字开头
            Book.objects.filter(sup__in=[100,200,300])              #   点赞量为100或200或300
            Book.objects.filter(dt__year__gt = 1999)                #   发行日期的年大于1999
            Book.objects.fileter(dt__lte = date(2002,4,15))         #   发行日期在2002.4.15之前
            Book.objects.filter(sup__gt=F("com"))                   #   点赞数大于评价数
            Book.objects.filter(Q(com__gt=120)&Q(sup__gt=130))      #   评论数大于120且点赞数大于130
            Book.objects.aggregate(Sum("sup"))                      #   和计点赞数
            Book.objects.all().order_by("-sup")                     #   按点赞数从大到小排
            Book.objects.order_by("-sup")                           #   按点赞数从大到小排
            Book.objects.order_by("com","-sup")                     #   先按照评论数从小到大，再按点赞数从大到小

    2.6.中间件(中间件函数是自动调用的无需手动调用)
        2.6.0.)django中间件工作过程:
            1)浏览器向django服务器请求--->2)django产生request对象--->3)django调用中间类中的process_request--->4)django进行url匹配
            --->5)django调用中间件中的process_view--->6)django调用view视图函数--->7)django调用中间件中的process_response
            --->8)django服务器将内容返回给浏览器
        2.6.1.)中间件的使用流程
            1)在应用中创建中间件文件(一般叫middleware.py)
            2)编写中间件类
                2.1。)
                    2.1.1.)中间件函数必须叫process_views;__init__;process_request;process_response;process_exception;否则不具备中间件的功能
                    2.1.2.)中间件类可以叫任意名字，但必须符合python命名规则
                2.2.)中间件的写法及作用
                    2.2.1.)
                        from django.utils.deprecation import MiddlewareMixin
                        class MiddleWare(MiddlewareMixin):              #   可以继承object或MiddlewareMixin
                            """中间件类"""
                            def __init__(self,get_response):
                                """服务器启动时，第一个请求调用"""
                                self.get_response = get_response        #   无此行极易出现非代码性bug
                                pass
                            def process_request(self,request):
                                """产生request对象之后，url匹配之前调用"""
                                pass
                            def process_view(self,request,view_fun,*view_args,**view_kwargs):
                                """url匹配后，视图函数调用之前调用"""
                                pass
                            def process_response(self,request,response):        #   还可以叫process_template_response
                                """视图函数调用之后，内容的返回"""
                                return ...                                      #   一般返回处理过后的response
                            def process_exception(self,request,exception):
                                """视图函数发生异常时调用"""
                                pass
                    2.2.2.)参数说明
                        request:request对象，包含请求的信息(相当于每个视图函数必须具备的request参数)
                        view_fun:视图函数
                        *view_args:视图函数位置参数
                        **view_kwargs:视图函数关键字参数
                        response:返回给浏览器的内容对象
                        exception:异常信息对象
            3)注册中间件类
                setting.py中修改MIDDLEWARE列表:
                    MIDDLEWARE = ["应用名.中间件文件夹名。中间件类名"]

                        url("^client_upload_html$",client_upload_html),     #上传文件的html

                    ]
                views.py:
                    from django.shortcuts import render
                    from django.conf import settings
                    from .models import *

                    #返会上传图片的html页面
                    #   /client_upload_html
                    def client_upload_html(request):
                        return render(request,"client_upload_html.html")

                    #保存用户图片的函数
                    #   /client_upload_save
                    def client_upload_save(request):
                        #1.获取上传文件的处理对象
                        img = request.FILES["client_img"]
                        #2.创建一个空文件用来保存浏览传来的数据
                        save_path = "{0}/{1}".format(settings.MEDIA_ROOT,"Img_Client/"+img.name)            #img.name:浏览器上传图片的名字
                        #3.获取浏览器上传文件内容并写入文件
                        with open(save_path,"wb") as f:
                            for content in img.chunks():                                                    #img.chunks():浏览器上传图片文件数据包
                                f.write(content)
                        #4.在数据库中保存上传记录
                        Img_Admin.objects.create(img="Img_Client/"+img.name)
                        #5.返回
                        return HttpResponse("上传成功!!!")

            from django.core.paginator import Paginator
            Paginator = Paginator(object_list=查询集,per_page=每页数据数)         #   Paginator对象,包含所有的页码
            Page = paginator.page(number=页码数)                                 #   page对象,只包含指定的页码
        2.9.2.)page类对象的属性和方法(在html模板和.py文件中均可使用):
            属性:
                number          --->    返回当前页码
                object_list     --->    返回包含当前页的数据的查询集
                paginator       --->    返回对应的Paginator类对象
            方法:
                has_previous            --->    判断当前页是否有前一页
                has_next                --->    判断当前页是否有下一页
                previous_page_number    --->    返回前一页的页码
                next_page_number	    --->    返回下一页的页码















通过shell操作数据表:
    python manage.py shell
    在shell中:
        import 应用名.models
        #1.   在表中添加 标记
        实例对象名 = 应用名.models.models总模板对象名()
        #2.   给标记的某个字段添加 特定的值
        实例对象名.实例对象的某个字段名 = 字段的值
        #3.   保存(把标记添加到表中)
        实例对象名.save()
        #4.   获取条件对象
        模型名.objects.get(条件)
        #5.   获取所有对象
        模型名.objects.all()
        #6.   通过外键获取对象
        模型名.objects.filter(外键名=某个外键)
        #7.   删除对象
        实例对象名.delete()
        #8.   退出shell
        exit()

    eg：
        在shell中输入
        #   应用名：djg_app_01  models：模板   BookInfo：models中的BookInfo模板   book1：实例对象

        import djg_app_01.models as model
        from datetime import date
        book1 = model.BookInfo()                    #创建实例对象(相当于表的标记)
        book1.bookname = "龙珠Z"                     #给book1标记的bookname字段赋值
        book1.bookdate = date(1999,1,3)             #给book1标记的bookdate字段赋值
        book1.save()                                 #保存book1标记
        BookInfo.objects.get(id=1)                    #获取BookInfo模型中id=1的对象
        book = BookInfo.objects.get(id=1)
        book.delete()                                 #删除id=1的对象
        exit()                                        #退出python manage.py shell
